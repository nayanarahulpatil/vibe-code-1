Every PRD MUST contain the following sections:
1. **Problem Statement**
   * What problem is being solved?
   * Who is affected?
   * Why is it important?
2. **Solution Overview**
   * High-level solution description.
   * Key modules/features.
3. **User Flow**
   * Main user journeys.
   * Happy path and failure path.
4. **API Design**
   * Endpoints, methods, requests, responses.
   * Authentication and error handling.
5. **Edge Cases**
   * Invalid inputs.
   * Failure scenarios.
   * Security and operational exceptions.
6. **KPIs (Success Metrics / Acceptance Criteria)**
   * Business KPIs.
   * Product KPIs.
   * Technical KPIs.
   * Feature acceptance criteria.
7. **Limitations**
   * Out-of-scope items.
   * Technical constraints.
   * Known risks and dependencies.
   
## Validation Rule
A PRD is considered **INCOMPLETE** if any of the above sections are missing.
